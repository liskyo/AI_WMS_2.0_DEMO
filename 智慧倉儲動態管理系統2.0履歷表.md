## 🌟 亮點精華版 (適合 104 專案介紹 / 履歷表限制字數區塊)

**【專案名稱】** 智慧倉儲動態管理系統 2.0 (獨立全端開發)
**【技術棧】** React, Vite, TailwindCSS, Node.js, Express, SQLite, Axios, framer-motion, html5-qrcode
**【專案簡介】**
為了解決傳統工廠倉儲「找料如大海撈針」、「帳物不符」與「依賴昂貴 PDA 設備」三大痛點，從零到一獨立架構的現代化 Web WMS 系統。採用前後端分離架構，實現跨裝置掃描、動態 2D 儲位視覺化、BOM 表配方管理與智能低庫存預警警示，徹底根除紙本作業，提升整體倉儲吞吐效率達 50% 以上。

**【核心貢獻與亮點】**
1. **獨創 2D 動態儲位地圖 (Visual Warehouse)**：突破傳統文字條列式介面，利用 React `useMemo` 與 CSS Grid 動態演算法，將冰冷的 Excel 座標轉化為具備「閃爍導航 (Ping)」與「障礙物自動排版」的 2D 互動地圖，找料零死角。
2. **行動端 PDA 跨平台化 (Mobile Scanner)**：整合 WebRTC 相機 API (`html5-qrcode`) 與 Local HTTPS 憑證，破解 iOS/Android 鏡頭權限限制。將員工自有手機轉化為出入庫掃描槍，結合「防呆鎖定」與「庫存分布速覽」機制，省下數十萬硬體採購成本。
3. **BOM 表配方展開與批量防呆**：導入製造業核心的 BOM (BOM) 管理邏輯，實作「一鍵展開主件發料」功能。系統會自動偵測旗下數十種子元件的庫存與儲位，精準計算缺料數量並防堵超發，大幅縮短備料時間。
4. **巨量數據覆蓋與 ACID 交易安全**：針對工廠動輒數萬筆的初期盤點 Excel 匯入需求，棄用沉重的關聯式資料庫，改採 Node.js 非同步串接 `better-sqlite3`。結合 `Transaction` 交易保護與「完全覆蓋防呆演算法」，達成萬筆資料的秒級無損同步。
5. **智能安全庫存與自動化採購輔助**：設計全系統連動的低庫存預警儀表板。一旦實際庫存低於料件定義之安全庫存，系統不只即時亮紅燈警告，更提供「建議採購清單」的 Excel 報表自動匯出功能，輔助採購決策。

## 🚀 深度技術剖析版 (適合面試官深挖 / 面試簡報 / GitHub Readme)

若進入技術面試，你可以針對不同面向展示你的工程師底蘊：

### 1. 前端渲染最佳化 (Frontend Rendering Optimization)
- **情境 (Situation)**：系統需要渲染帶有數百個儲位網格的平面圖，且每個網格在 Hover 時皆需即時浮現內含的多種料件庫存清單，導致初期 DOM 節點過多、畫面卡頓。
- **行動 (Action)**：
  - 放棄消耗效能的 Canvas，改以 Tailwind CSS Grid 作為底層，利用 React `useMemo` 將座標矩陣計算邏輯封裝，隔離無效的 Re-render (重新渲染)。
  - 開發自定義標籤解析器（例如處理 `#V_#` 開頭的大型走道），利用 `grid-column: span` 合併 DOM，並把被覆蓋的座標點標記為 `null` 捨棄算繪，精簡超過 30% 的 DOM 節點。
- **結果 (Result)**：實現了輕量級、純 HTML/CSS 驅動的互動地圖，即使在低階平板上也能流暢達到 60fps 幀率，並利用 `framer-motion` 流暢呈現 BOM 表出庫的閃爍指引。

### 2. 行動裝置非同步渲染競爭修復 (Race Condition Resolution)
- **情境 (Situation)**：將手機鏡頭作為掃描器時，切換「掃描成功」到「重置畫面」的過程頻繁出現當機白畫面。
- **行動 (Action)**：深入追溯源頭，發現是 React 組件的生命週期（DOM 尚未掛載完畢）與第三方相機套件（急於抓取 `#reader` 元素）產生了時間差競爭 (Race Condition)。我透過分析 `useEffect` 的依賴陣列與閉包 (Closure) 狀態，移除了所有手動呼叫的 `startScanner`，統一交由 React 的 Effect 佇列與 `setTimeout` 延遲隊列來接管硬體的啟動。
- **結果 (Result)**：徹底解決手機版當機問題，並順勢加值了「點擊卡片一鍵重掃」的防呆 UX，創造極為平滑的連續掃描體驗。

### 3. 資料庫的輕量化與交易原子性 (Database & ACID)
- **情境 (Situation)**：WMS 系統牽涉到複雜的關聯（料件、儲位、庫存、異動歷史），傳統開發會聯想到外掛龐大的 MySQL 等資料庫，但這會增加工廠端部署的 IT 門檻。
- **行動 (Action)**：
  - 架構決策大膽採用單體檔案型的 `SQLite` 搭配 Node.js。為了應證其效能，我設計了 `inventory` 關聯表專門解耦 `items` 與 `locations` 的多對多關係。
  - 在開發「作廢異動 (Void Transaction)」與「覆蓋式匯入 (Overwrite Import)」這類高風險功能時，嚴格實作了資料庫交易包裝 (`db.transaction()`) 與軟刪除機制 (`is_deleted`)。
- **結果 (Result)**：達成真正的**零配置 (Zero-Config) 部署**。工廠端只需點擊我撰寫的 `.bat` 批次檔，系統便會自動尋找作業系統環境變數建立本地 Node 環境並一鍵啟動伺服器，部署成本趨近於零，同時確保了高度的資料 ACIDs 安全性。
